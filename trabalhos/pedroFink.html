Cloud Computing - FATEC - 5o Semestre

EC2

Amazon Elastic Compute Cloud

O EC2 oferece máquinas virtuais sob demanda, mas seu diferencial real está na granularidade de controle: você escolhe arquitetura, tipo de CPU (inclusive ARM com Graviton), storage e rede. É ideal para workloads previsíveis ou aplicações legadas que não foram projetadas para cloud-native. Também permite estratégias avançadas como auto scaling baseado em métricas personalizadas e uso de instâncias spot para reduzir custos drasticamente.

IaaS Escalabilidade

S3

Amazon Simple Storage Service

Mais do que um simples armazenamento de arquivos, o S3 é um data lake altamente durável (11 noves de durabilidade). Ele permite versionamento, políticas de ciclo de vida e integração com eventos (event-driven). Muito usado como backend de aplicações serverless e para hospedar sites estáticos com distribuição global via CDN.

Storage Data Lake

Lambda

AWS Lambda

O Lambda permite executar código sem gerenciar servidores, mas seu verdadeiro poder está na arquitetura orientada a eventos. Ele funciona como “cola” entre serviços AWS, reagindo a mudanças em S3, filas SQS ou streams. Ideal para microsserviços altamente desacoplados, processamento assíncrono e automações com custo quase zero em baixa utilização.

Serverless Event-driven

RDS

Amazon Relational Database Service

O RDS abstrai a complexidade operacional de bancos relacionais, mas o destaque está na automação: backups, failover e replicação multi-AZ. Ele é ideal para sistemas que exigem consistência forte, como ERPs e sistemas financeiros, com suporte a engines como PostgreSQL e MySQL.

Banco Relacional Alta Disponibilidade

CloudFront

Amazon CloudFront

CDN global da AWS que reduz latência ao servir conteúdo a partir de edge locations. Mais do que cache, ele permite regras avançadas de distribuição, autenticação com Lambda@Edge e proteção contra ataques DDoS integrada com Shield.

CDN Performance

Route 53

Amazon Route 53

Serviço de DNS altamente disponível que vai além do básico: permite roteamento baseado em latência, geolocalização e health checks. Isso possibilita arquiteturas resilientes e distribuídas globalmente, direcionando usuários para a melhor instância disponível.

DNS Roteamento Inteligente

IAM

AWS Identity and Access Management

O IAM é o núcleo de segurança da AWS. Ele permite controle granular de permissões através de políticas baseadas em JSON. Um uso avançado envolve o princípio do menor privilégio e roles temporárias, essenciais para ambientes seguros e automatizados.

Segurança Controle de Acesso

DynamoDB

Amazon DynamoDB

Banco NoSQL totalmente gerenciado com latência de milissegundos. Seu diferencial está na escalabilidade horizontal automática e no modelo baseado em chave-valor/documento. Muito utilizado em aplicações de alta performance como jogos, IoT e sistemas com grande volume de leitura/escrita.

NoSQL Alta Performance

CloudWatch

Amazon CloudWatch

Ferramenta de observabilidade que coleta métricas, logs e eventos. Vai além de monitoramento básico, permitindo criação de alarmes inteligentes e automações (ex: escalar recursos automaticamente). Fundamental para práticas de DevOps e SRE.
